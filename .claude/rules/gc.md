# 가비지 컬렉션 (GC) 규칙

Claude는 코드 작업 중 아래 시점에 스스로 MD 파일을 점검하고 정리한다.

## 언제 실행하나

- 새 기능을 추가하거나 기존 파일을 대규모로 수정한 직후
- rules/ 파일의 내용이 실제 코드와 어긋나는 것을 발견했을 때
- 대화 세션이 길어져 컨텍스트가 누적될 때

## 점검 항목

### CLAUDE.md
- 60줄 초과 여부 → 초과하면 상세 내용을 rules/로 이동하고 한 줄 요약으로 대체
- 실제 파일 경로·함수명이 변경됐는데 MD가 구버전을 가리키는지 확인
- 더 이상 쓰지 않는 기술·패턴이 여전히 언급되는지 확인

### rules/*.md
- 코드베이스에서 이미 준수되고 있는 규칙은 예시를 줄여 간결하게 유지
- 규칙이 서로 충돌하거나 중복되면 하나로 합친다
- 삭제된 컴포넌트·파일을 참조하는 규칙은 즉시 수정

### memory/
- 더 이상 유효하지 않은 프로젝트 메모리(stale 상태)는 삭제
- 코드를 보면 바로 알 수 있는 내용(파일 구조, 변수명 등)은 메모리에 저장하지 않음

## 오염 신호 (이런 패턴이 보이면 정리)

- MD에 없는 패턴으로 코드가 작성되기 시작할 때
- 같은 규칙 설명이 두 파일에 걸쳐 중복될 때
- rules 파일 하나가 100줄을 넘길 때 → 분리 검토
