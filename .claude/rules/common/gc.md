# 가비지 컬렉션 (GC) 규칙

Claude는 코드 작업 중 아래 시점에 **자동으로** MD 파일을 점검하고 정리한다. 별도 요청 없이 실행한다.

## 언제 실행하나

- 새 기능을 추가하거나 기존 파일을 대규모로 수정한 **직후 항상**
- rules/ 파일의 내용이 실제 코드와 어긋나는 것을 발견했을 때 **즉시**
- 대화 세션이 길어져 컨텍스트가 누적될 때
- **런타임 오류가 발생했을 때** — 아래 심각도 기준에 따라 즉시 rules에 추가

## 런타임 오류 자동 추가 기준

오류를 수정한 직후, 아래 심각도가 **상**이면 즉시 해당 rules 파일에 추가한다.

### 심각도 판단 기준

| 심각도 | 조건 | rules 추가 여부 |
|---|---|---|
| 상 | 빌드는 통과하지만 서버/앱 시작 시 크래시 | **즉시 추가** |
| 상 | 특정 입력에서 예외 없이 데이터 손상·유실 | **즉시 추가** |
| 상 | 보안 취약점 (인증 우회, 데이터 노출) | **즉시 추가** |
| 중 | 특정 조건에서만 재현되는 런타임 오류 | 같은 패턴 2회 이상 시 추가 |
| 하 | UI 깨짐, 스타일 오류 | 추가 안 함 (일회성) |

### 추가할 내용 형식

```
// X — 이렇게 하면 [오류 이름]이 발생한다
잘못된 코드 예시

// O
올바른 코드 예시
```

원인 한 줄 설명을 반드시 포함한다.

## 점검 항목

### CLAUDE.md
- 60줄 초과 여부 → 초과하면 상세 내용을 rules/로 이동하고 한 줄 요약으로 대체
- 실제 파일 경로·함수명이 변경됐는데 MD가 구버전을 가리키는지 확인
- 더 이상 쓰지 않는 기술·패턴이 여전히 언급되는지 확인

### rules/*.md
- 코드베이스에서 이미 준수되고 있는 규칙은 예시를 줄여 간결하게 유지
- 규칙이 서로 충돌하거나 중복되면 하나로 합친다
- 삭제된 컴포넌트·파일을 참조하는 규칙은 즉시 수정

### memory/
- 더 이상 유효하지 않은 프로젝트 메모리(stale 상태)는 삭제
- 코드를 보면 바로 알 수 있는 내용(파일 구조, 변수명 등)은 메모리에 저장하지 않음

## 오염 신호 (이런 패턴이 보이면 정리)

- MD에 없는 패턴으로 코드가 작성되기 시작할 때
- 같은 규칙 설명이 두 파일에 걸쳐 중복될 때
- rules 파일 하나가 100줄을 넘길 때 → 분리 검토
